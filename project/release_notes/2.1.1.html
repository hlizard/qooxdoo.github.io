qooxdoo 2.1.1 (2013-02-05)

Summary

This is a regular patch release, which includes some important bug fixes but no feature enhancements over the previous release. It is fully backwards-compatible to qooxdoo 2.1. Nothing needs to be changed in the JavaScript code of your existing apps if they are based on the previous 2.1 releases.

Bugfixes

See the corresponding bugzilla query for the issues resolved for qooxdoo 2.1.1.

Core

  • Vendor specific CSS names are now handled correctly, also in IE. (#6959)

Data Binding

  • Make sure to set the object controller's model to null before removing all items from the form controller.
  • Call the converter in case the property chain for binding breaks.
  • Splice on data arrays can now handle remove and add of the same item.

qx.Website

  • Fixed stopping of JS based animation which are delayed.
  • Animations started with one collection can now be stopped with another collection.

qx.Desktop

UI Core

  • Fixed an issue with CSS placeholders for text fields. This fix is recommended for all Firefox browsers with a version equal or higher than 19 (#7104).

Form

  • Make sure 0 (and only 0) will be accepted as value when the input field is set to required.

Widgets

  • Prevent exception on mouse wheel selection of a select box.
  • Close all menus on scroll via mouse wheel to make sure the positioning does not break.
  • The preselected of the tables select box editor has the propper value preselected.

Virtual Widgets

  • Fixed scroll into view issue on VirtualTree when a model change happened (#7137)

Theming

  • Prevent name collisions of dynamically created decorators.
  • Fixed disabled theming for menubar buttons.

qx.Mobile

  • Carousel's scrollLoop did not work on swipe (#7018)
  • Enabling transitions between Master Pages (#7041)
  • Fix: Environment key "device.type" returns "tablet" for IE 10 (#7068)
  • Fix: Problem with qx.ui.mobile.dialog.Picker.setSelectedIndex() (#7029)
  • Fix: qx.ui.mobile.dialog.Picker only fire changeSelection on swipe (#7025)
  • Fix: qx.ui.mobile.form.List does not refresh after sort (#7087)
  • Optimized position of qx.ui.mobile.dialog.Popup near the borders (#7092)
  • Fix: Some mobile animations did not work for Firefox (#7134)
  • qx.Mobile Popups were displayed behind Blocker in some browsers (#7135)

Applications

Demobrowser

  • Added GitHub demo again (which showcases the qooxdoo REST wrapper and uses GitHub OAuth and the GitHub Gists API) and made it cross-browser capable. (6664)

Tooling

Generation

  • Reduced the amount of resources when building applications. (7045)

Code Formatting

  • Fix of various pretty-printing issues (switch-case formatting, statement parsing, trailing comments, 'keep-column', 'comment-cols', 'padding' options)

Migration

  • Fix bug in the migration that could potentially corrupt code. The new implementation uses pretty-printing (with default options), so your code will be re-formatted. This is only relevant for users that haven't already migrated to 2.1.
