qooxdoo 1.5.1 (2011-12-07)

Summary

This is a regular patch release, which includes many bugfixes but no feature enhancements. It is meant to be backwards compatible to qooxdoo 1.5. Nothing needs to be changed in the JavaScript code of your existing apps if they are based on the previous 1.5 release.

Bugfixes

See the corresponding bugzilla query for about 150 issues resolved for qooxdoo 1.5.1.

Framework

Browser Support

  • More reliably detect arrays in Rhino environments. (#5109)

Core

  • Various improvements of the finite state machine. (#4940, #4941)
  • ObjectRegistry runs in DOM-less environments. (#5231)
  • Remove browser dependencies in qx.util.Base64. (#5282)

Data Binding

  • Several fixes in the marshaler. (#5223) (#5455) (#5401)
  • Fixes in the stores. (#5669) (#5770)

Low-level

  • Minor fix for the HTML5 Media wrapper. (#5837)
  • Minor fix for the HTML Geolocation wrapper. (#5836)
  • Add FileReader wrapper. (#4991)
  • Fixed wrong implementation of setHeight in qx.bom.media.Video. The functions now sets the height correctly. (#5760)
  • Fixed endless loop for invalid date formats. (#5521)
  • Fixed incorrect month localization format. (#5593)
  • Add security notice to API documentation of methods accepting HTML strings that are assigned to innerHTML. (#5481)

Transport

Low-level Transport

  • Cancel timeout before invoking handlers because they may throw. (#5831)
  • Fire onload event for successful requests when using the file protocol. (#5586)
  • Fix ontimeout() always being called for sync requests. (#5629)
  • Fix status for conditional requests in Opera. (#5626)
  • Leave status unchanged for aborted requests. (#5626)
  • Fix wrong handlers being called when aborting. (#5579)

High-level Transport

  • Make content-type to parser mapping more robust, accept parameters. (#5475)
  • Fix "fail" event fired twice for timeout or network error. (#5771)
  • Not check HTTP method since custom methods may be used (e.g. when using Webdav). (#5606)
  • Not parse empty response. (#5626)
  • Fix intermediate phases when aborting request. (#5485)
  • A user-defined content-type takes precedence, even when HTTP method is POST. (#5432)
  • Allow issuing standard JSON-RPC Version 2 requests from qx.io.remote.Rpc (experimental).

REST

  • Create new request for each invocation of action. (#5511)
  • Support parameter value that evaluates to false. (#5506)

GUI Toolkit

  • Support for OSX Lion scroll bars. (#5441)
  • Don't scroll container automatically when using radio buttons. (#5327)
  • Fix auto-size text area when enabling wrapping in IE7. (#4976)
  • Fix inheritance hierarchy of DateField. (#3161)
  • Fix input event of textfield fired on every keystroke. (#5309)

Widgets

  • The tables's image cell renderer now allows scaling. (#5047)
  • Minor fix in the menu. (#5419)
  • Make the iframe usable when hidden / not added to a parent. (#5418)

Theming

  • Minor corrections in the themes. (#5448) (#5610) (#5408)
  • Invalid themes are now recognized and an error is shown to the developer. (#5727)
  • Fixed color property for fonts. (#5368)

Applications

Showcase

  • Fix focus handling. (#5450)

Playground

  • Fixed cut-off text field in TreeVirtual filter demo. (#5643)

API Viewer

  • Fixed toggling "Mixin included". All methods included by a mixin are now shown and hidden correctly. (# 5461)
  • Selecting the right section when following a search result. (# 5451)

Test Runner

  • MMock: Make fake abort() switch state to DONE when OPENED and sent.

Feature Configuration Editor

  • Fix select-all key combo for Mac OS by listening to the "keydown" event.

Tooling

Generation

  • Fixed issue where deleted resource files would be looked for, e.g. during resource copying.

Internationalization

  • Normalize alternative Unicode representations of characters in file names (e.g. as direct code point or with combining character).

Migration

Manual Migration Steps

  • Configuration: Users working with the basic skeleton will need to edit their config.json when upgrading to 1.5.1: Please add "qx.bom.element.Overflow" to both the jobs/basic-common/config-warnings/exclude and jobs/basic-common/exclude keys.
