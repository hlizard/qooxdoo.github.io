qooxdoo 1.4.1 (2011-05-20)

Summary

This is a regular patch release, which includes bugfixes but no feature enhancements. It is meant to be backwards compatible to qooxdoo 1.4. Nothing needs to be changed in the JavaScript code of your existing apps, if they are based on the previous 1.4 release.

Bugfixes

For this out-of-schedule release only the following bugfixes were applied, in order to keep the testing efforts to a minimum while delivering the most significant improvements:

  • Unnecessary wrapping of rich labels in FF4 (#4961)
  • Liveupdate of text fields doesn't work in IE9 non-standards modes (#4982)
  • Fix PHP files vulnerabilities (#4977)
  • qx.Bootstrap complains "qxvariants is not defined" (#4996)
  • Variant keys with "on"/"off" values are not correctly evaluated during optimization (#5083)
  • qx.core.Environment.get("os.version") is not returning the correct result (#5014)

Also see the corresponding bugzilla query for issues resolved for qooxdoo 1.4.1.
