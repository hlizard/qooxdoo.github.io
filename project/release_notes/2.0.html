qooxdoo 2.0 (2012-06-21)

Summary

This is a new major release, which comes with many new features and improvements besides a large number of bugfixes.

Despite being a major release, only a limited number of manual migration steps may be necessary for upgrading from the previous 1.6.x releases. See the Migration section to determine if your apps are affected. As with most releases there might also be deprecation warnings in the development version, but those shouldn't negatively affect the production version of your app. It is nevertheless suggested to get rid of those warnings early. If you need a bugfix-only drop-in replacement, you might want to check-out qooxdoo 1.6.1, the corresponding patch release.

Bugfixes & Enhancements

See this detailed list of more than 350 issues resolved for qooxdoo 2.0, which includes the fixes of the accompanying qooxdoo 1.6.1 patch release.

Besides fixes for defects, many enhancements made it into qooxdoo 2.0. See here for a detailed list of issues that were resolved for 2.0 exclusively.

Core

  • Changed Environment keys (covered by migration job):
    • qx.disposerDebugLevel → qx.debug.dispose.level
    • qx.propertyDebugLevel → qx.debug.property.level
    • qx.ioRemoteDebug → qx.debug.io.remote
    • qx.ioRemoteDebugData → qx.debug.io.remote.data
  • Add support for namespace-less classes, i.e. classes with a bare name, without a "." in it, like "Foo". Mind that this increases the risk of name clashes across libraries. (experimental)
  • New LibraryManager provides read/write access to library-specific information, e.g. resource URIs
  • Added stacktrace properties to BaseError, GlobalError and WindowError for compatibility with qx.dev.StackTrace.getStackTrace.
  • Scripts will be loaded in parallel but executed in order in browsers that support the script element's "async" attribute.
  • Fixed Array handling in qx.lang.Object.toUriParameter.
  • Replace IE's incomplete Error.toString with qx.lang.Core.errorToString.
  • Fixed event data on qx.data.Array. (#6154)
  • Removed warning for images loaded via data URL.
  • Commands don't override themed icons anymore. (#5906)
  • Serializer processes inherited properties now. (#5559)
  • If an assertion check throws an error, the message will include responsible arguments with values of undefined as 'undefined'. (#6282)
  • New environment key "qx.blankpage" for blank.html resource implemented. All statements in framework using blank page are now bound to this key. (#6332)

Event

  • Added minimalistic event emitter class. (#4983)
  • Added a routing class. (#4795)

Data Binding

  • Fixed support for multiple mixins for the JSON marsahler. (#6445)
  • Added the possibility to change the property names in your created models. (#5907)
  • Added property and item to the changeBubble event. (#5942)
  • sort() method of qx.data.Array was firing events (change, changeBubble) although nothing has changed on the array. In this case we are now preventing these events.(#6213)

Browser Support

  • Feature detection for CSS border-image syntax to cope with differences between prefixed and non-prefixed/spec-compliant implementations.
  • Fixed animation for IE10.
  • Fixed gradient calculation for IE10. (#6513)
  • Cleand up storage implementation and added IE fallback. (#49)

qx.Website

  • Added support to build RequireJS packages. (#6041)
  • Added a reverse method for animations. (#2371)
  • Added JavaScript Fallback for CSS3 animations. (#6068)
  • Added a tutorial. (#6418)
  • Added a dedicated API viewer for qx.Website. (#6314)
  • Added support for IE8 to qx.bom.IframeHistory. Even though IE8 can natively use "hashchange" events, it fails if qooxdoo runs in an iframe. Therefore we need to poll a helper iframe like in in IE6/7 (#6194)(#6521)
  • Support for textShadow in qx.bom.Font implemented. This property can be used in themes. The value can be any valid string like specified in MDN. Currently there is no fall back for IE browsers. (#5554)
  • If running qooxdoo in iframe, iOS Safari just provides "orientationChange" event and window.orientation property only on top frames. Therefore the default window object is now window.top in qx.event.handler.Orientation and qx.bom.Viewport. But this can cause a same-origin policy exception if the top frame is not in same domain as the qooxdoo app. (#5554)

qx.Desktop

  • qx.ui.core.queue.Widget.add() can be called with a job identifier. You can check for this identifier in the .syncWidget(jobs) method of qx.ui.core.Widget (#6336)
  • In some cases it is useful to destroy a container widget and its children recursively. qx.util.DisposeUtil.destroyContainer(container) can handle that. (#6306)

UI Core

  • Added events for width and height changes. (#6385)
  • The rgbToHexString method now returns the hex6 string including the leading "#" character. (#6316)
  • Added spread radius (#6216) and inset (#6192) property for box shadow decorator.
  • Improved overall scroll handling.
  • qx.ui.core.Blocker fires now two new events: "blocked" and "unblocked" (#6351)

Form

  • Form now supports the virtual select box. (#5711)

Layouts

  • Adding any object which is not a qx.ui.core.LayoutItem now throws an error which points you to that issue (#6281)

Widgets

  • It's now possible to access the instance from an error tooltip. This opens some new ways to coustomize for example the validation. (#6309)
  • Properly disable resizing for disabled widgets e.g. windows. (#6516)
  • qx.ui.table.Table is now scrollable by touch events. (#6367)

Virtual Widgets

  • Fixed infinity loop for one selection when the last item from the VirtualList is removed. (#6340)
  • Nested RadioGroupBox are now correct disabled when parent is disabled. (#5445)

Theming

  • Added support for styling modal windows differently than non-modal ones (#6363)
  • Make themed fonts aware of colors from the color theme. (#6496)
  • Table shows its disabled state. (#5880)
  • Added gradient fallback for IE. (#6193)
  • Changed the look of the tabview in the simple theme.
  • Added new Theme named Indigo. (#6233)
  • The error popup now can have the arrow on the right side if the popup is shown right to the widget. (#6308)

qx.Mobile

  • Removed experimental status.
  • Tablet device support, out of the box.
    • API change for qx.ui.mobile.page.NavigationPage. NavigationPages have to be added to qx.ui.mobile.page.Manager before show() is called.
    • Added qx.ui.mobile.container.MasterDetail for tablet layout view. (#6381)
    • Added qx.ui.mobile.page.Manager as a convenience layer for tablet/mobile layout distinction. (#6381)
  • Added qooxdoo mobile theme (Indigo). (#4800)
  • Added theme switcher to qx.Mobile showcase. (#5955)
  • Many styling improvements in mobile themes (iOS, Android and Indigo)
  • Split up theme.less to widgetXY.less → refactored mobile theming. (#6383)
  • qx.ui.mobile.form.ToggleButton is now able to display custom state text. (#6453)
  • New qx.ui.mobile.form.CheckBox and qx.ui.mobile.form.RadioButton style for Android and Indigo theme, optimized for mobile usage (same appearance on desktop/mobile, bigger touch area).
  • Added device-specific qx.ui.mobile.form.ToggleButton styling (Android/iOS). (#5393)
  • New widget qx.ui.mobile.form.NumberField (#5603)
  • Added qx.ui.mobile.dialog.Menu widget. (#6237)
  • Added qx.ui.mobile.form.RadioGroup. (#6294)
  • Added own qx.ui.mobile.form.SelectBox implementation (same appearance on all browser/devices). (#6054)
  • Added device.type environment variable for device class detection (Desktop,Tablet or Mobile) (#6392)
  • Added Basic and Data Binding Page to qx.Mobile showcase. (#6304)
  • Fixed unwanted page scrolling during text input focus, when iScroll is used. (#6300)
  • Toolbar buttons shows text ellipsis. (#5873)
  • Mobile tests can be executed in Mozilla Firefox. (#6122)
  • Added hasCssClass() and toggleCssClass() to qx.ui.mobile.core.Widget. (#6219) (#6379)

qx.Server

Library

  • Renamed library files (to comply with qx.Website):
    • qx-oo.js → qx-oo.min.js
    • qx-oo-noopt.js → qx-oo.js

Applications

Portal

  • The Portal app has been turned into a Demobrowser demo, and removed from the standard apps.

API Viewer

  • Class and method documentation can now be enhanced with automatically generated links to an SCM's web view, e.g. GitHub.

Test Runner

  • The Widget and HTML views will now display results of performance tests (using qx.dev.unit.MMeasure). The specialized Performance view is deprecated.
  • Test definitions for the Portable Testrunner support custom namespaces to improve the readability of the results list.
  • Configuration options (such as macro definitions or environment settings) defined in the "common" job are now applied to the application under test (AUT).
  • Unit tests can now be skipped by calling this.skip().

Skeletons

  • Renamed skeletons to comply with component names (this affects creating new apps):
    • bom → website
    • basic → server
    • gui → desktop

Tooling

Generation

  • New parser for compiles, build output formatting and api data extraction.
  • The compile cache will use less disk space, as cache objects are now compressed.
  • New generate.py command line option –no-progress-indicator to suppress rotating dash progress indication; this is interesting for batch runs which are logged.
  • The api job now warns about missing package docs (which go into __init__.js files).

Configuration

  • compile-options/code/optimize: New optimization keys [ "whitespace", "comments" ] to activate whitespace (compression) and comment removal individually (both are on by default).
  • compile-options/paths/scripts-add-hash: This key is deprecated, as script names (except for the loader script) contain a hash part anyway.
  • translate/poentry-with-occurrences: Allows you to disable occurrences comments in .po files (comments of the kind "#: qx/locale/Key.js:80" that list the occurrences of a key in the code); we felt they don't provide much help but create a lot of noise when you revision-control .po files, and after a generate.py translation only changed locations with otherwise unchanged keys would result in a diff. Still on by default.
  • log/dependencies/phase: Allow null value to include both run-time and load-time dependencies.
  • combine-images/montage-cmd: New key exposes the ImageMagick montage command line.
  • slice-images/convert-cmd: New key exposes the ImageMagick convert command line.
  • New ADD_NOCACHE_PARAM config macro, to tailor compile-options/uris/add-nocache-param.
  • compile-options/paths/file-prefix: New key to add the contents of a file at the top of generated script files (e.g. a copyright header).
  • generate.py now ignores commented-out QOOXDOO_PATH entries in config.json.
  • generate.py now honors -c/–config command-line options (so particularly will use QOOXDOO_PATH from the given configuration file).

Optimization

  • Support for optimizing logical expression during variant optimization, like in if (qx.core.Environment.get("foo") && qx.core.Environment("bar")).
  • The generator now warns if both variants optimization and the qx.allowUrlSettings environment setting are enabled in the same build, as you then can only influence settings through URL parameters that are not set as variants in the config.
  • In build versions of an app, format is now false by default (i.e. no extra line breaks will be inserted in the optimized code); re-enable with compile-options/code/format:true.

Internationalization

  • The Norwegian language code no has been removed from the CLDR data. Please use nb ("Norwegian Bokmål", which "no" used to redirect to) or nn ("Norwegian Nynorsk") instead. "NO" as territory code is ok (as in "nb_NO").
  • Updated CLDR data to 21.0.1.

Documentation

  • Major overhaul of the manual to reflect the new components (qx.Website, qx.Desktop, qx.Mobile, qx.Server).
  • New manual page about custom libraries.

Migration

  • Configuration: If you have overridden settings of the OPTIMIZE macro or compile-options/code/optimize config key you need to add [ "whitespace", "comments" ] if you want to retain those optimizations.
  • Deprecated the qx.fx.* animation layer in favor of qx.bom.element.Animation (CSS3 animations + JavaScript fallback). (#6075)

API Changes

  • qx.util.ColorUtil.rgbToHexString() returns the hex string with a leading "#".
  • qx.bom.Storage takes the role of removed (experimental) classes qx.bom.storage.(Abstract|Local|Session).

Missing Features / Known Issues

  • The support to inspect mobile applications with the Inspector is broken. (#6480)
